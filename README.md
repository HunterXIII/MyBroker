# Message Broker

Собственный брокер сообщений с поддержкой модели **Publish/Subscribe**, очередями сообщений и базовой персистентностью.

Проект реализован на языке **Go** и разработан в учебных целях для изучения архитектуры систем обмена сообщениями и принципов работы брокеров.

---

# Цели проекта

Основная цель проекта — реализовать упрощённую версию брокера сообщений и изучить следующие аспекты:

- архитектуру систем обмена сообщениями  
- модель **publish / subscribe**  
- обработку сетевых соединений  
- конкурентность в Go (goroutines и channels)  
- работу с очередями сообщений  
- персистентное хранение данных  
- доставку сообщений подписчикам  
- мониторинг и метрики  

Проект демонстрирует базовые принципы работы таких систем как:

- Apache Kafka  
- RabbitMQ  
- NATS  
- MQTT brokers  

---

# Основные возможности

Брокер поддерживает базовый набор функций:

- подключение клиентов к серверу  
- публикация сообщений в топики  
- подписка клиентов на топики  
- доставка сообщений подписчикам  
- очередь сообщений для каждого топика  
- гарантия доставки **at-least-once**  
- сохранение сообщений в **append-only лог**  
- восстановление состояния после перезапуска  
- экспорт метрик для **Prometheus**

---

# Архитектура

Брокер реализован как **модульный монолит**.

Это означает, что система запускается как одно приложение, но внутри разделена на независимые логические модули.

Основные компоненты системы:

- **Network Layer** — управление TCP соединениями  
- **Protocol Layer** — обработка клиентских команд  
- **Broker Core** — центральная логика брокера  
- **Topic Manager** — управление топиками  
- **Queue Manager** — очереди сообщений  
- **Delivery Engine** — доставка сообщений подписчикам  
- **Storage Layer** — персистентность сообщений  
- **Monitoring Layer** — сбор и экспорт метрик

Преимущества:

- поддержка модели pub/sub  
- компактные сетевые пакеты  

---

## Персистентность

Используется **append-only лог сообщений**.

Сообщения записываются последовательно в файл: `data/messages.log`

Такой подход обеспечивает:

- высокую скорость записи  
- простое восстановление состояния  
- минимальные накладные расходы хранения  

---

## Мониторинг

Используется **Prometheus**.

Метрики доступны через HTTP endpoint: `/metrics`

---

## Контейнеризация

- Docker  
- docker-compose  

для запуска брокера и вспомогательных сервисов.

# Запуск проекта

## Клонирование репозитория

```
git clone https://github.com/your-username/message-broker.git  
cd message-broker
```

---

## Сборка

```
go build ./cmd/broker
```

---

## Запуск

```
./broker
```

По умолчанию брокер запускает TCP сервер и принимает подключения клиентов.

# Метрики

Метрики доступны по адресу:


- количество доставленных сообщений

- количество топиков  
- размер очередей
